WTF is this

Graphics information exposed by the WebGL API is unreliable and doesn't give you an approximation of performance. We record the GL renderer name and match it with benchmark results. The more people use the benchmark, the more accurate we'll be.

Quickstart

Install the library

$ npm install --save benchmark.gl

Use

const bmgl = require("benchmark.gl");

const performanceQuotient = bmgl.calibrate({
    preset: 'graphics' // or 'compute', 'shader'
});

That's it.
